Sentier vs Autoroute

Philosophie architecturale qui guide tous les choix techniques du projet.

🛣 L'Autoroute

  • Infrastructure standardisee GAFAM
  • Consommation de solutions pre-packagees
  • Dependance vendor (MCP = lock-in)
  • Atrophie cognitive
  • Rapide mais superficiel

🌳 Le Sentier

  • Approche artisanale du code
  • "Ping-pong contre un mur" (iteration haute frequence)
  • Modeles locaux, RAG souverain
  • Friction comme moteur de croissance
  • Lent mais profond (marathon cognitif)
"Garder l'aventure plutot que le resultat" — Choisir le Sentier. Accepter la friction du "ping-pong contre un mur", les exigences de l'artisanat du code, et la responsabilite de la coordination humaine.

Lichtung — Pre-Reflexion Obligatoire

Systeme de pre-reflexion architecturale qui force la pensee deliberee (System 2) avant toute action significative. Nomme d'apres le concept heideggérien de "la clairiere" ou la verite peut apparaitre.

"La dette technique nait non pas du code casse, mais du code fonctionnel qui est architecturalement dissonant. La Lichtung existe pour empecher cela."

defaultMode: "plan"

Couche 2 : Agent
Agent Claude Opus avec outils volontairement desactives (Edit, Write, Bash). Techniquement incapable d'ecrire du code.
Couche 3 : NILE Skill
/lichtung "query"

Protocole en 5 phases

Phase 1 : Exploration
Grep, Glob, Read — JAMAIS deviner, TOUJOURS verifier

"Tous les X", "classifier", "indexer" → proposer script plutot qu'action manuelle
Phase 2 : Questionnement (Regle des 3)
Q1 Impact, Q2 Contraintes, Q3 Patterns — minimum 3 questions avant d'agir
Phase 3 : Planification
Plan structure : fichiers impactes, etapes, risques identifies, mitigations
Phase 4 : Validation
